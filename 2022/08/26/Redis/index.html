Redis

Redis安装

ubuntu安装redis

sudo apt update
#安装redis服务
sudo apt install redis-server
#重启redis服务
sudo systemctl restart redis-server.service

Redis设置密码

sudo vim /etc/redis/redis.conf
#查找requirepass，将password改为自己的密码，保存退出，重启服务
requirepass password

Redis开启远程链接

sudo vim /etc/redis/redis.conf
# 查找bind 127.0.0.1；将其注释或者改为如下
bind 0.0.0.0

Redis-cli连接

//无密码连接
redis-cli -h host -p port
//如：redis-cli -h localhost -p 6379

//有密码连接
redis-cli -h host -p port -a password
//如：redis-cli -h localhost -p 6379 -a "password"

问题【Redis密码连接】：Warning: Using a password with ‘-a’ or ‘-u’ option on the command line interface may not be safe.

//解决方式一: --no-auth-warning参数
//不进行验证警告
redis-cli -h localhost -p 6379 -a "password" --no-auth-warning

//解决方式二: 无密码登录后使用auth password验证
redis-cli -h localhost -p 6379
auth password

Springboot集成Redis

  • 导入依赖
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-redis</artifactId>
    <version>2.7.3</version>
</dependency>
  • 设置Redis信息
spring:
  redis:
    host: localhost
    port: 6379
    password: YB520lly..
    timeout: 3000
    jedis:
      pool:
        max-active: 10
        max-wait: -1
        max-idle: 10
        min-idle: 0
    database: 0
  • 创建RedisConfig
@Configuration
public class RedisConfig {

    @Value("${spring.redis.host}")
    private String host;

    @Value("${spring.redis.password}")
    private String password;

    @Value("${spring.redis.database}")
    private Integer database;

    @Value("${spring.redis.port}")
    private Integer port;

    @Bean
    @ConfigurationProperties(prefix = "spring.redis.jedis.pool")
    public JedisPoolConfig jedisPoolConfig() {
        return new JedisPoolConfig();
    }

    @Bean
    public RedisConnectionFactory redisConnectionFactory(JedisPoolConfig jedisPoolConfig) {
        RedisStandaloneConfiguration redisStandaloneConfiguration = new RedisStandaloneConfiguration();
//        设置主机、端口、数据库、验证密码等
        redisStandaloneConfiguration.setHostName(host);
        redisStandaloneConfiguration.setPort(port);
        redisStandaloneConfiguration.setDatabase(database);
        redisStandaloneConfiguration.setPassword(password);

        JedisClientConfiguration.JedisPoolingClientConfigurationBuilder jedisPoolingClientConfigurationBuilder
                = (JedisClientConfiguration.JedisPoolingClientConfigurationBuilder) JedisClientConfiguration.builder();
        jedisPoolingClientConfigurationBuilder.poolConfig(jedisPoolConfig);
        JedisClientConfiguration jedisClientConfiguration = jedisPoolingClientConfigurationBuilder.build();

//        创建Jedis连接工厂
        return new JedisConnectionFactory(redisStandaloneConfiguration, jedisClientConfiguration);
    }

    @Bean
    public RedisTemplate<String, Object> redisTemplate(RedisConnectionFactory redisConnectionFactory) {
        RedisTemplate<String, Object> redisTemplate = new RedisTemplate<>();
        redisTemplate.setConnectionFactory(redisConnectionFactory);
//        配置redis的key序列化方式
        StringRedisSerializer stringRedisSerializer = new StringRedisSerializer();
        redisTemplate.setKeySerializer(stringRedisSerializer);
        redisTemplate.setHashKeySerializer(stringRedisSerializer);
//        配置redis的value序列化方式
        Jackson2JsonRedisSerializer<Object> jackson2JsonRedisSerializer = new Jackson2JsonRedisSerializer<>(Object.class);
        redisTemplate.setValueSerializer(jackson2JsonRedisSerializer);
        redisTemplate.setHashValueSerializer(jackson2JsonRedisSerializer);

        redisTemplate.afterPropertiesSet();
        return redisTemplate;
    }

//    封装RedisTemplate，可省略
    @Bean
    public OBRedisClient obRedisClient(RedisTemplate<String, Object> redisTemplate){
        return new OBRedisClientImpl(redisTemplate);
    }

}

常用设置

配置 说明
daemonize (on|yes) 使用 yes 启用守护进程，默认为no
pidfile path/file.pid 以守护进程方式运行时，Redis 默认会把 pid 写入/var/run/redis/redis-server.pid（默认） 文件，可以通过 pidfile 指定。
port 端口号 默认端口为 6379
bind ip 绑定的主机地址,如果指定为 0；表示关闭该功能,允许任何机器访问
timeout time 当客户端闲置多长秒后关闭连接，如果指定为 0 ，表示关闭该功能
loglevel type 指定日志记录级别，Redis 总共支持四个级别：debug、verbose、notice、warning，默认为 notice
logfile stdout 日志记录方式，默认为标准输出，如果配置 Redis 为守护进程方式运行，而这里又配置为日志记录方式为标准输出，则日志将会发送给 /dev/null
databases num 设置数据库的数量，默认数据库为0，可以使用SELECT 命令在连接上指定数据库id
requirepass password 设置 Redis 连接密码，如果配置了连接密码。关闭使用config set requirepass 即可。

Redis数据类型

类型 说明
字符串（string） String 是一组字节。在 Redis 数据库中，字符串是二进制安全的。这意味着它们具有已知长度，并且不受任何特殊终止字符的影响。可以在一个字符串中存储最多 512 兆字节的内容。
哈希（hash） 哈希是键值对的集合。在 Redis 中，哈希是字符串字段和字符串值之间的映射。因此，它们适合表示对象。
列表（list） Redis 列表定义为字符串列表，按插入顺序排序。可以将元素添加到 Redis 列表的头部或尾部。
集合（set） 集合（set）是 Redis 数据库中的无序字符串集合。在 Redis 中，添加，删除和查找的时间复杂度是 O(1)。
有序集合（sorted set） Redis 有序集合类似于 Redis 集合，也是一组非重复的字符串集合。但是，排序集的每个成员都与一个分数相关联，该分数用于获取从最小到最高分数的有序排序集。虽然成员是独特的，但可以重复分数。
位图 ( Bitmaps ) Redis Bitmap 通过类似 map 结构存放 0 或 1 ( bit 位 ) 作为值。
基数统计 ( HyperLogLogs ) Redis HyperLogLog 可以接受多个元素作为输入，并给出输入元素的基数估算值

Key操作

Keys pattern：查找指定pattern的key

//返回所有匹配pattern的key；*表示返回所有的key
keys pattern
//keys str?
//keys *
匹配模式
符号 说明 例子
? 匹配一个字符 str?匹配：str1、stra等等
* 匹配任意长度字符 str*匹配：str、str1、str11等等
[ab…] 匹配括号中任意一个字符，匹配多个字符无效 str[12]匹配：str1、str2
[^ab…] 匹配不包含括号中任意一个字符，匹配多个无效，^必须放在开头，否则无效 str[^12]不匹配：str1、str2
[a-b] 配置包含a到b范围内的任意一个字符 str[1-3]匹配：str1、str2、str3

Sorted Set：有序集合

Sorted Set介绍

  • string类型的有序集合
  • 集合成员唯一
  • 每个元素都会关联一个 double 类型的分数，通过分数来为集合中的成员进行从小到大的排序
  • 分数 ( score ) 可以重复。
  • 增删查时间为O(1)
  • 最大容量：(2^32) - 1个
