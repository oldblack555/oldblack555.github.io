Sa-Token介绍

Sa-Token 是一个轻量级 Java 权限认证框架，主要解决：登录认证、权限认证、单点登录、OAuth2.0、分布式Session会话、微服务网关鉴权 等一系列权限相关问题。

SpringBoot集成Sa-Token

建议：jdk>=1.8；springboot>=2.0

implementation 'cn.dev33:sa-token-spring-boot-starter:1.31.0'

Application.yml配置token信息（可选，不配置则使用默认配置）

sa-token:
  #  token 名称
  token-name: sa_token
  #  token 超时时间
  timeout: 60
  #  是否允许多个登录
  is-concurrent: true
  #  是否共享一个token
  is-share: false
  #  token风格
  token-style: uuid
  #  是否输出日志
  is-log: false
  # token 临时有效期 (指定时间内无操作就视为token过期) 单位: 秒
  activity-timeout: -1

简单登录认证

主要使用StpUtil类中方法，实现核心是通过Cookie进行Token认证。

StpUtil.login(id)：登录生成Token

@RequestMapping("/login")
public SaResult login(String username, String password) {
    if (USER_MAP.containsKey(username) && password.equals(USER_MAP.get(username))) {
        String id = UUID.randomUUID().toString().replaceAll("-", "");
        log.info("[ {} id is {} ]", username, id);
        //核心步骤，登录成功后会在Cookie设置Token，前端只要保存并在发起请求时带Token的Cookie，后端会进行登录判断。
        //该方法的具体原理暂不介绍
        StpUtil.login(id);
        return SaResult.ok("LOGIN SUCCESS");
    }
    return SaResult.ok("LOGIN FAILED");
}

StpUtil.isLogin()：通过Cookie携带的Token登录判断

@RequestMapping("/login/check")
public SaResult checkLogin() {
    //登录检查，
    //方式一：会对请求中Cookie的Token进行验证，
    //方式二：自定义header，tokenname = token
    //登录返回true，否则返回false
    return SaResult.ok("是否登录: " + StpUtil.isLogin());
}

也可以使用StpUtil.checkLogin()进行判断，但是在未登录时会抛出NotLoginException。

StpUtil.checkLogin()：检查用户是否登录，未登录抛出异常，通常检查登录的状态（登录、强制下线、被踢等），如果只需要结果使用前面的方法即可

StpUtil.checkLogin();

StpUtil.getLoginIdDefaultNull()：获取登录ID

@RequestMapping("/login/id")
public SaResult getLoginId() {
    //获取登录的ID，
    //方式一：会对请求中Cookie的Token进行验证，
    //方式二：自定义header，tokenname = token
    //已登录返回登录ID，否则返回NULL
    return SaResult.data("当前登录ID: " + StpUtil.getLoginIdDefaultNull());
}

StpUtil.getTokenTimeout()：获取登录有效时长

@RequestMapping("/login/token/timeout")
public SaResult getTokenTimeout() {
    //获取登录有效时长，
    //方式一：会对请求中Cookie的Token进行验证，
    //方式二：自定义header，tokenname = token
    //已登录返回登录有效时长，-1表示永久登录，否则返回-2表示未登录
    return SaResult.data("当前token时间: " + StpUtil.getTokenTimeout());
}

StpUtil.getTokenInfo()：获取Token信息

@RequestMapping("/login/token/info")
public SaResult getTokenInfo() {
    //获取Token详细信息，
    //方式一：会对请求中Cookie的Token进行验证，
    //方式二：自定义header，tokenname = token
    //已登录返回登录详细信息，否则返回未登录的Token状态
    return SaResult.data("当前token信息: " + StpUtil.getTokenInfo());
}

StpUtil.logout()：退出登录

@RequestMapping("/logout")
public SaResult logout() {
    log.info("当前注销的token is {}", StpUtil.getTokenInfo());
    //退出登录，
    //方式一：会对请求中Cookie的Token进行验证，
    //方式二：自定义header，tokenname = token
    //如果Token已登录，则会清楚相关信息。
    StpUtil.logout();
    return SaResult.ok("LOGOUT SUCCESS");
}

Http请求样例

# 登录
GET http://localhost:8082/login?username=yb&password=123456
# 验证
# 方式一：自定义header，携带tokenname（服务器设置的token名称）和value
GET http://localhost:8082/login/check
Content-Type: application/json
sa_token: e26904c7-7a77-4e0f-b7e1-37f14323447d
# 方式二：使用登录成功返回的Cookie进行提交
GET http://localhost:8082/login/check
Content-Type: application/json
Cookie: sa_token=785b9582-9b4e-4c44-9662-457a61598381

简单权限认证

实现StpInterface接口

StpInterface提供id和权限、角色绑定方法。用户登录时，通过实现的StpInterface接口，将用户id和数据库或者其他提供用户权限、角色进行list的自动绑定。

@Component
public class PermissionAuthImpl implements StpInterface {

    // 对用户id和加载的权限列表绑定，限定用户的权限
    @Override
    public List<String> getPermissionList(Object loginId, String loginType) {
        // 本list仅做模拟，实际项目中要根据具体业务逻辑来查询权限
        List<String> list = new ArrayList<>();
        list.add("101");
        list.add("user-add");
        list.add("user-delete");
        list.add("user-update");
        list.add("user-get");
        list.add("article-get");
        return list;
    }

    // 对用户id和加载的角色列表绑定，限定用户的角色
    @Override
    public List<String> getRoleList(Object loginId, String loginType) {
        // 本list仅做模拟，实际项目中要根据具体业务逻辑来查询角色
        List<String> list = new ArrayList<>();
        list.add("admin");
        list.add("super-admin");
        return list;
    }
}

        log.info("[ {} id is {} ]", username, id);
        //核心步骤，登录成功后会在Cookie设置Token，前端只要保存并在发起请求时带Token的Cookie或者自定义的header，后端会进行登录判断。
        //该方法的具体原理暂不介绍
        //login方法会自动绑定角色和权限。
        StpUtil.login(id);

权限认证

前提：http请求中需要携带有token的Cookie或者自定义的header中tokenname和value

// 获取：当前账号所拥有的权限集合
StpUtil.getPermissionList();

// 判断：当前账号是否含有指定权限, 返回true或false
StpUtil.hasPermission("user-update");        

// 校验：当前账号是否含有指定权限, 如果验证未通过，则抛出异常: NotPermissionException 
StpUtil.checkPermission("user-update");        

// 校验：当前账号是否含有指定权限 [指定多个，必须全部验证通过]
StpUtil.checkPermissionAnd("user-update", "user-delete");        

// 校验：当前账号是否含有指定权限 [指定多个，只要其一验证通过即可]
StpUtil.checkPermissionOr("user-update", "user-delete");

角色认证

前提：http请求中需要携带有token的Cookie或者自定义的header中tokenname和value

// 获取：当前账号所拥有的角色集合
StpUtil.getRoleList();

// 判断：当前账号是否拥有指定角色, 返回true或false
StpUtil.hasRole("super-admin");        

// 校验：当前账号是否含有指定角色标识, 如果验证未通过，则抛出异常: NotRoleException
StpUtil.checkRole("super-admin");        

// 校验：当前账号是否含有指定角色标识 [指定多个，必须全部验证通过]
StpUtil.checkRoleAnd("super-admin", "shop-admin");        

// 校验：当前账号是否含有指定角色标识 [指定多个，只要其一验证通过即可] 
StpUtil.checkRoleOr("super-admin", "shop-admin");

权限通配符：*****

使用权限统配符可以简化匹配规则（角色也试用），如

user* 匹配 user-list、user-add、user-delete等

* 匹配所有的角色和权限。

强制下线

方式一：强制注销

StpUtil.logout(10001);                    // 强制指定账号注销下线 
StpUtil.logout(10001, "PC");              // 强制指定账号指定端注销下线 
StpUtil.logoutByTokenValue("token");      // 强制指定 Token 注销下线

方式二：踢人下线

StpUtil.kickout(10001);                    // 将指定账号踢下线 
StpUtil.kickout(10001, "PC");              // 将指定账号指定端踢下线
StpUtil.kickoutByTokenValue("token");      // 将指定 Token 踢下线

注解方式权限认证

配置连接器

@Configuration
public class SaTokenConfigure implements WebMvcConfigurer {
    // 注册 Sa-Token 拦截器，打开注解式鉴权功能 
    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        // 注册 Sa-Token 拦截器，打开注解式鉴权功能 
        registry.addInterceptor(new SaInterceptor()).addPathPatterns("/**");    
    }
}

权限检查注解（可类可方法）

// 登录认证：只有登录之后才能进入该方法 
@SaCheckLogin

// 角色认证：必须具有指定角色才能进入该方法 
@SaCheckRole("super-admin")

// 权限认证：必须具有指定权限才能进入该方法 
@SaCheckPermission("user-add")

//校验模式
//SaMode.AND, 标注一组权限，会话必须全部具有才可通过校验。
//SaMode.OR, 标注一组权限，会话只要具有其一即可通过校验。
@SaCheckPermission(value = {"user-add", "user-all", "user-delete"}, mode = SaMode.OR)

//角色权限双重 “or校验”
//写法一：orRole = "admin"，代表需要拥有角色 admin 。
//写法二：orRole = {"admin", "manager", "staff"}，代表具有三个角色其一即可。
//写法三：orRole = {"admin, manager, staff"}，代表必须同时具有三个角色。
@SaCheckPermission(value = "user-add", orRole = "admin") 

// 二级认证：必须二级认证之后才能进入该方法 
@SaCheckSafe() 

// Http Basic 认证：只有通过 Basic 认证后才能进入该方法 
@SaCheckBasic(account = "sa:123456")

// 校验当前账号是否被封禁 comment 服务，如果已被封禁会抛出异常，无法进入方法 
@SaCheckDisable("comment") 

//可以游客访问 
@SaIgnore
